Skills & Technologies Used


Skills

  • Time Management
  • Waterfall Model Design
  • Identifying the Client's Requirements
  • Creating Navigation Diagram
  • Creating Website Wireframes
  • Developing Prototypes
  • Self Evaluation of Designs
  • Conducting a Cognitive Walkthrough
  • Adapting to Feedback

Technologies

  • GIMP was used for brainstorming and planning (Rich Picture).
  • Draw.io was used to create diagrams.
  • Google Slides was used to create website wireframes, and website prototypes.
  • Bootstrap was the CSS framework of choice. It allowed me to create a responsive, and mobile-first website.
  • PHP was used to store and retrieve information from a database
  • PostgreSQL was the database of choice.
  • GitLab was used for source control.


Development Screenshots


Rich Picture

At the start of the project, I made a rich picture (below) to visualise the client's requirements and how each stakeholder will interact with the website.

Use Case Diagram

The use case diagram was developed from the previous rich picture. The use case diagram shows the areas which the stakeholders have access to on the website.

Navigation Diagram

The navigation diagram incorporates all points identified in the rich picture and use case diagram. With this information, I can make a basic structure of the website and start to think about the UI design.

Wireframe Design

The wireframe design below is the homepage design. This also includes the menu design which will be used throughout the website. Multiple other wireframes were made for the rest of the website before moving to the prototype.

Prototype

The prototype was created using Google Slides. It uses hyperlinks to simulate website navigation. With the prototype made, I conducted a cognitive walkthrough to gather feedback about the design.

The final interactive prototype can be viewed/downloaded from here.

Updated Design

Following feedback from the cognitive walkthrough, I started to create a live website with appropriate design changes. This allowed me to improve the design further.

The final website can be found here.

Group Project



Project Summary

For the group project, we were put into random groups of five and set the task of creating an online booking registration system.
My initial role within the group was to create an in-depth test plan to ensure sure that the website functioned as intended. After I completed the test plan, I quickly adapted to working with the front-end development team to create responsive web pages.
